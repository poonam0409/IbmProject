Description

Runs Copyrights On Demand (COD) against a set of files. COD assists product teams in complying with the requirements of IBM Corporate Standard C-S 0-6045-002 by automating the maintenance of the copyright banners within its source parts.

Also See

  • How to enable local use of the COD Ant task
  • rtc-changes Ant task
  • User's Guide
  • Visit the COD community

Parameters

Attribute Description Required
check

Controls if the files are just checked (read-only) or actually updated (read-write).

  • If false, any file that requires updating is re-written
  • If true, each file is processed and any errors are reported but no file is re-written
 No; defaults to false
defs Defines the directory housing the COD definition files. Yes
failonerror

Controls what happens when processing error(s) are detected:

  • If false, log a warning message, but do not stop the build <default>
  • If true, log a warning message, and stop the build.
 No; defaults to false.
verbose Provide extra information during processing. No; defaults to false.
year Define the copyright year to be used. Valid values are the previous, current, and next year. No; defaults to the current year.

Parameters specified as nested elements

fileset

One or more <fileset> elements are used to select the files to processed.

The <date> selector provides a powerful means of processing just the set of files modified in the current release. 

  <date datetime="08/27/2007 12:00 AM" when="after" />
If interested in RTC source repository support, see the rtc-changes Ant task.

Examples

Update a set of files 

  <cod defs="../defs" >
    <fileset dir="../src" />
  </cod>

Check a set of files 

  <COD defs="../defs" check="true">
    <fileset dir="../src" />
  </COD>

Update a set of files stopping the build if error(s) are detected

  <cod defs="../defs" failonerror="true" >
	<fileset dir="../bin" includes="**/*.bat, **/*.sh" />
	<fileset dir="../doc" />
	<fileset dir="../src" />  
  </cod>
