Introduction

The COD tool assists product teams in complying with the requirements of IBM Corporate Standard C-S 0-6045-002.

It automatically updates a product's source parts with the appropriate copyright banners. Traditionally, this was put off until the end of the product development cycle and was then accomplished through a "blitz" of all parts. Perhaps home grown scripts were used to guess the location of banners based on text searches. More often than not, the approach was time consuming and/or error prone.

By using the Copyrights On Demand tool and making minor changes to business practice, a development team can effectively update banners on server al hundreds of files in seconds rather than days.

COD is written in Java, so it can used on most development platforms. Over its lifetime, it has been developed and tested on various Windows and Linux platforms.

History

  • COD was originally developed by Joseph Firebaugh and Steve Diering to automatically process the thousands of source parts that make up the Tivoli Business System Manager (TBSM) product.
  • The legacy COD versions (v1.0 to v3.0) were driven by the desires of IBM and Tivoli product development teams who embraced COD as part of their development process. These versions were housed in the IIOSB Community Source repository.
  • Starting with v4.0, COD has been hosted housed in the CSNext Community Source repository.

What's New in v4.0.0b? [06/24/2011]

Migration from IIOSB to CSNext

  • The COD project was re-created in the CSNext infrastructure
  • All new development/enhancements will be performed from this location

Shift to an Ant-centric Implementation

  • These previously available functions have been removed:
    1. CLI interface
    2. Swing UI interface
    3. Maven support [since based on CLI support]
    4. Eclipse plug-in
  • All of these remain available from the legacy project

COD Ant Task

New

  • Added cod.format attribute to product.properties that allows the format of the updated copyright years to be controlled

Changes

  • The summary statistics now accurately reflect the number of files that would be updated when the check attribute is used

rtc-changes Ant Task

New

  • This task provides support for RTC based source repositories
  • The COD task can then use the rtc-changes task output file to define the files to be processed
