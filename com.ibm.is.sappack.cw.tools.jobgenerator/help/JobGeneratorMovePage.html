Where indicated, you may refer to a job parameter in the form #parameter_name#, e.g., #DayOfWeek#. When you actually use one of the parameters from a parameter set in your job design, you refer to it in the form #parameter_set.parameter_name#. For example, if you want your job to use the value of the parameter DayOfWeek from the parameter set Runschedule, you refer to it in the job as #Runschedule.DayOfWeek#. For more information on parameters and parameter sets, please see the WebSphere DataStage and QualityStage Designer Client Guide.

  • Model File: The model file describing the source/target schema. Note, that both model files have to be equal with respect to the contained entities and their attributes.
  • ODBC DataSource: The data objects will be read/written through this datasource. The ODBC datasource has to be configured in the operating system. (#<identifier># defines a job parameter)
  • ODBC User: (optional): Overwrites the user in the datasource configuration. (#<identifier># defines a job parameter)
  • ODBC Password: (optional): Overwrites tha password in the datasource configuration. (#<identifier># defines a job parameter)
  • Reject File Server Path: (optional): The generated job will log all rejected objects in this file if specified
