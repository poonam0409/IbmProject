IDoc import options

Use this page to configure how IDoc types are imported into a logical data model. Each segment is mapped to a relational table and additional fields are used to establish the parent and child relationship of the IDoc segment hierarchy.

IDoc Segment fields
Specify how different types of IDoc segment fields are mapped to attributes in the logical data model:
Make all segment fields nullable
Sets the nullable flag on all fields.
Allow all types for all columns
The types of the attributes are derived from the data types of the segment fields such as VARCHAR or DATE.
Use VARCHAR only
Use VARCHAR as the type for all attributes. If this option is selected, you can define a length factor by which the original length is multiplied. This option is useful if you want to use the resulting tables to temporarily hold data whose length exceeds the maximum length defined by SAP.
Check table options
Specify options to import related check tables that contain valid values for certain fields.
Import metadata for related check-tables
Import all check tables for the imported IDoc segment fields.
Import metadata for related text-tables
Import the related text table for every check table. These text tables contain descriptions of values of check tables.
Check table blacklist
A file containing list of check tables to avoid during the check table import. Specify one table name per line.
