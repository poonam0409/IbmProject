ABAP Extract 選項

使用此頁面，可以在工作產生設計中配置 ABAP Extract 選項。

註： 使用 "#" 符號括住文字，就可以針對文字欄位使用 InfoSphere™ DataStage® 工作參數。 此參數集是以語法 "#" #ParameterSetName.ParameterName# 表示。如需參數及參數集的相關資訊，請參閱《IBM® InfoSphere DataStage 及 QualityStage™ Designer 用戶端手冊》。
來源選項
DataStage SAP 連線
DataStage 中配置的 SAP 連線名稱。
SAP 使用者
要在連接至 SAP 時使用的 SAP 使用者 ID。
SAP 使用者密碼
SAP 使用者的密碼。
SAP 使用者用戶端號碼
SAP 使用者的用戶端號碼。
SAP 使用者語言
SAP 使用者的語言。
目標選項
暫置類型
產生為 SAP 邏輯表格目標的暫置類型。 例如，ODBC Connector 或 Sequential File。
ODBC Connector
指定 ODBC Connector 的這些選項：
ODBC 資料來源
從 SAP 擷取的物件會儲存並配置在 ODBC 資料來源表格中。
ODBC 使用者
改寫資料來源配置中的使用者。這是選用欄位。
ODBC 密碼
改寫資料來源配置中的密碼。這是選用欄位。
選取表格動作
您可以選取 APPEND 或 TRUNCATE 表格動作。如需相關資訊，請參閱 ODBC Connector 暫置文件。
Sequential File
指定 Sequential File 暫置的這些選項：
檔名字首
用來儲存輸出的所有檔案的字首。
檔案更新模式
您可以選取 APPEND、CREATE（如果存在則發生錯誤）或 OVERWRITE。如需相關資訊，請參閱 Sequential File 暫置文件。
第一行是直欄名稱
如果您要循序檔中的第一行包含直欄名稱，請選取此選項。
RFC 傳送方法選項
指定 RFC 傳送方法選項：
SAP 閘道主機
與 SAP 伺服器通訊的 SAP 閘道供應項目服務的主機名稱。
SAP 閘道服務
閘道服務的名稱。
RFC 目的地
簡式文字檔，每行都包含兩個以空格區隔的項目。第一個項目是 SAP 系統上所配置的 RFC 目的地名稱。第二個項目是 RFC 目的地的對應程式 ID。
工作中的流程數上限
您可以使用多個 ABAP 暫置來產生 ABAP Extract 工作，以減少工作數目。不過，在 ABAP Extract 工作中，每個 ABAP 暫置都需要在 SAP 中配置唯一的 RFC 目的地。因此，ABAP 暫置數上限受限於提供的 RFC 目的地數目。它也會定義每個工作的流程數上限，或單一 IBM InfoSphere DataStage 工作內的 ABAP 暫置實例（受限於提供的 RFC 目的地數目）。
自動建立 RFC 目的地
如果要由 IBM InfoSphere DataStage 工作在執行時期建立 RFC 目的地，請選取此選項。
必要時刪除 RFC 目的地
選取此選項，可以允許在必要時刪除 RFC 目的地。
驗證 SAP 設定
按一下此按鈕，以驗證 SAP 連線設定。
