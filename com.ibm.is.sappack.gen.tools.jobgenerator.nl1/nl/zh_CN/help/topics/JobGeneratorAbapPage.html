IBM InfoSphere Information Server Pack for SAP Applications V7.0
反馈

ABAP 抽取选项

使用该页面可配置作业生成设计中的 ABAP 抽取选项

注： 通过使用“#”符号括起文本，可以对文本字段使用 InfoSphere™ DataStage® 作业参数。使用语法“#”指示参数集：#ParameterSetName.ParameterName#。有关参数和参数集的更多信息，请参阅 IBM® InfoSphere DataStage and QualityStage™ Designer 客户机指南
源选项
DataStage SAP 连接
DataStage 中配置的 SAP 连接的名称。
SAP 用户
连接 SAP 时要使用的 SAP 用户的标识。
SAP 用户密码
SAP 用户的密码。
SAP 用户客户机编号
SAP 用户的客户机编号。
SAP 用户语言
SAP 用户的语言。
目标选项
阶段类型
作为 SAP 逻辑表目标生成的阶段的类型。例如，ODBC Connector 或 Sequential File。
ODBC 连接器
指定 ODBC 连接器的以下选项：
ODBC 数据源
从 SAP 抽取的对象存储在已配置的 ODBC 数据源表中。
ODBC 用户
覆盖数据源配置中的用户。该字段是可选的。
ODBC 密码
覆盖数据源配置中的密码。该字段是可选的。
选择表操作
您可以选择 APPEND 或 TRUNCATE 表操作。有关更多信息，请参阅 ODBC Connector 阶段文档。
Sequential File
指定 Sequential File 阶段的以下选项：
文件名前缀
用于存储输出的所有文件的前缀。
文件更新方式
您可以选择 APPEND、CREATE（存在时出错）或 OVERWRITE。有关更多信息，请参阅 Sequential File 阶段文档。
第一行为列名称
选择是否希望顺序文件中的第一行包含列名称。
RFC 传输方法选项
指定 RFC 传输方法选项：
SAP 网关主机
与 SAP 服务器通信的 SAP 网关产品服务的主机名。
SAP 网关服务
网关服务的名称。
RFC 目标
一个简单文本文件，其中每一行包含用空格分隔的两个条目。第一个条目是 SAP 系统上配置的 RFC 目标的名称。第二个条目是 RFC 目标所对应的程序标识。
作业中的最大流数
可以生成包含多个 ABAP 阶段的 ABAP 抽取作业，以减少作业数量。但是，在 ABAP 抽取作业中，每个 ABAP 阶段都需要在 SAP 中配置一个唯一的 RFC 目标。因此，ABAP 阶段的最大数量受所提供的 RFC 目标数量的限制。它还定义每个作业的最大流数或者单个 IBM InfoSphere DataStage 作业中的 ABAP 阶段实例数量，该数量受所提供的 RFC 目标数量的限制。
自动创建 RFC 目标
如果希望在运行时通过 IBM InfoSphere DataStage 作业创建 RFC 目标，请选择该选项。
删除 RFC 目标（如果需要）
如果需要，选择该选项可删除 RFC 目标。
验证 SAP 设置
单击该按钮可验证 SAP 连接设置。
