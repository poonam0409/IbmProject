ABAP 추출 옵션

이 페이지에서는 작업 생성 디자인에서 ABAP 추출 옵션을 구성할 수 있습니다.

주: 텍스트를 "#" 기호로 둘러싸 텍스트 필드에 대해 InfoSphere™ DataStage® 작업 매개변수를 사용할 수 있습니다. 매개변수 세트는 "#" #ParameterSetName.ParameterName# 구문으로 표시됩니다. 매개변수 및 매개변수 세트에 대한 자세한 정보는 IBM® InfoSphere DataStage and QualityStage™ 디자이너 클라이언트 안내서의 내용을 참조하십시오.
소스 옵션
DataStage SAP 연결
DataStage에서 구성된 SAP 연결의 이름입니다.
SAP 사용자
SAP에 연결할 때 사용할 SAP 사용자의 ID입니다.
SAP 사용자 비밀번호
SAP 사용자의 비밀번호입니다.
SAP 사용자 클라이언트 번호
SAP 사용자의 클라이언트 번호입니다.
SAP 사용자 언어
SAP 사용자의 언어입니다.
대상 옵션
스테이지 유형
SAP 논리 테이블의 대상으로 생성되는 스테이지의 유형입니다. ODBC Connector 또는 순차 파일을 예로 들 수 있습니다.
ODBC Connector
ODBC Connector를 위한 다음 옵션을 지정하십시오.
ODBC 데이터 소스
SAP에서 추출된 오브젝트는 ODBC 데이터 소스 테이블에 저장 및 구성됩니다.
ODBC 사용자
데이터 소스 구성의 사용자를 겹쳐씁니다. 이 필드는 선택적입니다.
ODBC 비밀번호
데이터 소스 구성의 비밀번호를 겹쳐씁니다. 이 필드는 선택적입니다.
테이블 조치 선택
APPEND 또는 TRUNCATE 테이블 조치를 선택할 수 있습니다. 자세한 정보는 ODBC Connector 스테이지 문서를 참조하십시오.
순차 파일
순차 파일 스테이지에 다음 옵션을 지정하십시오.
파일 이름 접두부
출력을 저장하는 데 사용되는 모든 파일의 접두부입니다.
파일 업데이트 모드
APPEND, CREATE(존재하는 경우 오류) 또는 OVERWRITE를 선택할 수 있습니다. 자세한 정보는 순차 파일 스테이지 문서를 참조하십시오.
첫 번째 행은 열 이름
순차 파일의 첫 행에 열 이름을 포함하려는 경우 선택
RFC 전송 방법 옵션
RFC 전송 방법 옵션을 지정하십시오.
SAP 게이트웨이 호스트
SAP 서버와 통신하기 위한 서비스를 제공하는 SAP 게이트웨이의 호스트 이름입니다.
SAP 게이트웨이 서비스
게이트웨이 서비스의 이름입니다.
RFC 대상
각각의 행이 공백으로 구분되는 두 개의 항목으로 구성되는 간단한 텍스트 파일입니다. 첫 번째 항목은 SAP 시스템에서 구성되는 RFC 대상의 이름입니다. 두 번째 항목은 RFC 대상의 해당 프로그램 ID입니다.
작업 내 최대 플로우 수
작업 수를 줄이기 위해 여러 ABAP 스테이지로 ABAP 추출 작업을 생성할 수 있습니다. 하지만, ABAP 추출 작업에서 각 ABAP 스테이지는 SAP에서 고유한 RFC 대상을 구성해야 합니다. 따라서 ABAP 스테이지의 최대 개수는 제공되는 RFC 대상의 수에 의해 제한됩니다. 또한, 이 수는 단일 IBM InfoSphere DataStage 작업 내에서 ABAP 스테이지 인스턴스 또는 작업당 플로우의 최대 개수를 정의하며, 제공되는 RFC 대상의 수로 제한됩니다.
RFC 대상 자동 작성
런타임에서 IBM InfoSphere DataStage 작업에 의해 RFC 대상이 작성되는 경우 이 옵션을 선택하십시오.
필요한 경우 RFC 대상 삭제
필요한 경우 RFC 대상 삭제를 허용하려면 이 옵션을 선택하십시오.
SAP 설정 유효성 검증
SAP 연결 설정을 유효성 검증하려면 이 단추를 클릭하십시오.
