トラブルシューティング

このトラブルシューティングのトピックでは、IBM® InfoSphere™ Information Server Pack for SAP Applications で検出される可能性がある問題を取り扱います。

クライアント

Rapid Modeler for SAP または Rapid Generator for SAP の使用中に問題が検出される場合、ワークスペースの .metadata ディレクトリーで .log ファイルを調べて、エラーがないか確認します。

サーバー

Rapid Generator for SAP の場合、サーバー・コンポーネントのエンタープライズ・アプリケーションをインストールした WebSphere® Application Server でログ・ファイルを確認したい場合もあります。このディレクトリーは通常、<WAS install directory>/AppServer/profiles/default/logs/server1 にあります。SystemOut.log ファイルおよび SystemErr.log ファイルから始めます。 トレースを使用可能にした場合、trace.log ファイルも表示されます。

診断トレースをオンにするには、WebSphere Application Server ヘルプのそれぞれのページを参照してください。トレース・コンポーネントの名前は com.ibm.is.SAPPacks です。

例

IBM InfoSphere Information Server Pack for SAP Applications に関連したいくつかの例があります。 これらの例には、既存のデータ設計プロジェクトが必要です。

これらの例を作成するには、「ファイル」 > 「新規」 > 「例」 > 「IBM InfoSphere Information Server Pack for SAP Applications」に進みます。
  • ビジネス・オブジェクトのサンプル・モデル: これは、Rapid Modeler for SAP で使用できる、事前定義された SAP モデルの例です。
  • Rapid Modeler for SAP で使用されるチェック・テーブル・ブラックリストのサンプル・ファイル。 Rapid Modeler for SAP では、インポート時に無視されるテーブルの名前が入っているファイルを使用できます。『InfoSphere Data Architect Rapid Modeler for SAP を使用した SAP メタデータのインポート』を参照してください。これは、このファイルがどのように表示されるかを示す例です。
  • ABAP Extract ジョブのために Rapid Generator for SAP で使用される RFC 宛先用のサンプル・ファイル。

    RFC 転送方式を使用する ABAP Extract ステージを使用してジョブを生成するには、RFC 宛先とそれに関連したプログラム ID のリストをファイルで与える必要があります。これは、このファイルがどのように表示されるかを示す例です。

既知の問題

このセクションには、既知の問題の説明が記載されています。InfoSphere DataStage® における IBM InfoSphere Information Server Pack for SAP R/3 の既知の問題とヘルプについては、Pack for SAP R/3, Version 6.5.0.1: Known problems を参照してください
  1. 問題: 生成された IDoc Load または Extract ジョブが、次のエラー・メッセージのいずれかを出して終了する。
    • 致命的エラー: Fatal: IDOC_F_00131`:`Required property 'DEFAULTUSERNAME' not found in configuration file.
    • 致命的エラー: Fatal: IDOC_F_00131`:`Required property 'SAPAPPSERVER' not found in configuration file.

    解決方法: 与えられた接続名が存在し、DataStage プロジェクト用に正しく構成されていることを確認してください。

  2. 問題: SAP JCo ライブラリーをロードできない。

    解決方法: IBM technote 1418488 を参照してください。この技術情報は、別の製品用に作成されましたが、解決方法はここでも適用されます。

  3. 問題: Rapid Generator for SAP が、IBM DB2® Database for Linux, UNIX, and Windows 以外の物理モデル用のジョブを生成しない。

    理由: 現在、Rapid Generator for SAP では、DB2 for Linux/Unix/Windows に対しては物理データ・モデルしかサポートされていません。
