概要

ほとんどすべての SAP プロジェクトの一部として、SAP のデータ・モデルを識別し、分析し、文書化する必要があります。これらのデータ・モデルはほとんどの場合、カスタマイズされ、補完されているので、カスタム・データ・モデルは標準モデルとは異なっています。SAP システムからデータを抽出したり、SAP システムにデータをロードしたりするには、外部アプリケーションと SAP インターフェースに関する知識が必要です。企業内では一般に、異なるチームがデータ統合と SAP システムをそれぞれ担当します。この責任の分割の結果、(例えば、フラット・ファイルの交換に) 一時の間に合わせで統合が実行されることになります。

外部アプリケーションと SAP システムを統合する場合、次の 2 つの課題が生じます。

  • このタスクに必要なデータは、SAP システムのどこにあるのか? SAP 内のどのデータ構造にどの情報が保管されているのか?
  • そのデータを外部アプリケーションとの間で、どのように抽出したりロードしたりするのか? どのような抽出手段やロード手段が使用可能であり、それらを使用するにはどうしたらいいか?

これらの課題は、互いに無関係ではありません。例えば、SAP IDoc はデータ構造と転送メカニズムの両方を定義します。 転送メカニズムは、SAP 論理テーブルを抽出するときに通常使用されるメカニズムと同じではありません。 以下のツールが、これらの問題に対処します。

IBM® InfoSphere™ Data Architect Rapid Modeler for SAP Applications
このツールを使用すると、SAP データ・モデルを参照し、IBM InfoSphere Data Architect を使用して作成される外部キー関係により、関連したすべてのテーブルをインポートすることができます。インポートするテーブルからダイアグラムを作成したり、メタデータをよりよく理解するためのコメントや注釈を追加したりすることができます。
IBM InfoSphere Rapid Generator for SAP Applications
このツールを使用すると、IBM InfoSphere Data Architect Rapid Modeler for SAP Applications を使用してインポートした SAP メタデータを使用して InfoSphere DataStage® ジョブを 生成できます。これらの InfoSphere DataStage ジョブは、 インポートされたメタデータによって記述された SAP オブジェクトの実際のデータを、リレーショナル・データベース またはフラット・ファイルから抽出したり、それらにロードしたりします。
フラット・ファイル、リレーショナル・データベース、またはその他の SAP システムなどの各種ソース・システムから、統合された単一の SAP システムに、データを移動することができます。この処理の間、ステージング・エリア と呼ばれることがある中間データベースを作成します。このデータベースを使用すると、ターゲット・システムにロードされるすべてのデータを保管できます。さらに、複雑な正規化や洗浄の操作がここで実行されます。IBM InfoSphere Data Architect Rapid Modeler および IBM InfoSphere Rapid Generator for SAP を使用すると、以下を行うことができます。
  • SAP ターゲット・システムのステージング・エリアのスキーマを作成する。
  • ステージング・エリアからターゲット SAP システムをロードするジョブを生成する。
  • スキーマを作成し、SAP ソース・システムの抽出ジョブを生成する。

IBM InfoSphere Data Architect Rapid Modeler for SAP Applications ツールと IBM InfoSphere Rapid Generator for SAP Applications ツールの両方に、InfoSphere Data Architect の拡張 (Eclipse プラグイン) として実装されるクライアントがあります。 また、両方のツールが、SAP システムへの接続を提供する SAP JCo ライブラリーに依存します。IBM InfoSphere Rapid Generator for SAP Applications は、InfoSphere DataStage ジョブを生成し、コンパイルするために InfoSphere Information Server が必要です。エンタープライズ・アプリケーションは、InfoSphere Information Server が使用するのと同じ WebSpehere Application Server のインスタンスに、.ear ファイルとしてデプロイされます。

図 1. IBM InfoSphere Information Server Pack for SAP Applications のアーキテクチャー
  • IBM InfoSphere Data Architect Rapid Modeler for SAP Applications の概要
    IBM InfoSphere Data Architect Rapid Modeler for SAP Applications は、SAP システムから IBM InfoSphere Data Architect の論理データ・モデルにメタデータを抽出するウィザードです。
  • IBM InfoSphere Rapid Generator for SAP Applications の概要
    IBM InfoSphere Rapid Generator for SAP Applications は、SAP との間でデータの抽出やロードを行う InfoSphere DataStage ジョブを InfoSphere Data Architect 内の物理データ・モデルから生成するウィザードです。 この物理データ・モデルは、IBM InfoSphere Rapid Modeler for SAP を使用して、SAP メタデータをインポートした論理データ・モデルを変換することによって、作成する必要があります。
