Reconciling your model with an SAP system

After you imported SAP metadata into a logical data model, the metadata represents a snapshot of the SAP system at that moment. Sometimes you might want to re-import or reconcile your model by running InfoSphere™ Rapid Modeler for SAP metadata import wizards again on a new model. After the new model is created, you can use InfoSphere Data Architect to compare the original and the new logical data model. The metadata re-import recollects all check and text tables.

Before you begin

Ensure that the SAP metadata was imported into a logical data model by using the InfoSphere Rapid Modeler.

About this task

Some of the scenarios where it is useful to reconcile metadata are:
  • Compare the logical data model with the model in the same SAP system to see if it changed. For example, the model might change after upgrading to a new version or after a development cycle.
  • Compare one SAP system with another. For example, you might find differences between a test system with the corresponding production system or between two separate SAP systems.

Procedure

  1. Right-click the logical data model (.ldm file) that contain SAP metadata, and select Rapid Modeler for SAP > Re-import metadata into new model.
  2. On the Previous importer runs page, select the previous import runs performed with the InfoSphere Rapid Modeler for SAP, and click Next.
  3. Enter the name of the new logical data model to be created without the .ldm file extension.
  4. Select the SAP system you want to reimport metadata from.
  5. Click Start Re-import.
  6. ClickOK. The InfoSphere Rapid Modeler for SAP import wizard starts containing the options of the original import.
  7. To compare the two logical data models, right-click the model, then select Compare with > Each other.

Results

A new logical data model is created that contains the SAP metadata that was reconciled.
