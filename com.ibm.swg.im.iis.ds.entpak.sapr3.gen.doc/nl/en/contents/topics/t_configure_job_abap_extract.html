Configuring job generation for ABAP extract jobs

You can use the InfoSphere™ Rapid Generator for SAP Applications wizard to generate InfoSphere DataStage® jobs that extract ABAP data from an SAP system. Every IBM® InfoSphere™ DataStage® job using the RFC data transfer method must be configured to use a unique RFC destination. You can run only one ABAP extract job at a time as ABAP stage using the same RFC destination cannot run concurrently.

Before you begin

  • Configure your workspace for use with InfoSphere Rapid Modeler for SAP and InfoSphere Rapid Generator for SAP.
  • Configure the InfoSphere DataStage environment must be configured correctly.
  • Import SAP metadata into a logical data model, and transform the logical data model into a physical data model.

Procedure

  1. Right-click the Physical data model (.dbm file), and select Rapid Generator for SAP > Generate DataStage jobs.
  2. On the Select Job Type page, select ABAP Extract.
  3. On the Information Server Details page, enter the connection information to your InfoSphere Information Server installation and provide a prefix for all the jobs that are created when the wizard is run. Make sure that the connection details are the same as the ones entered when using the InfoSphere Information Server client applications such as the InfoSphere DataStage and QualityStage™ Designer client.
  4. Click Next to validate the connection details.
  5. In Source options, specify details about the SAP system from which you want to extract logical tables.
  6. In Target options, configure stages generated as the target of the SAP logical tables.
  7. On ABAP extract options, specify how the ABAP extract jobs are created.
  8. Click Validate SAP settings to validate all SAP-related settings on this page. The validation result is displayed in a new window. Continue with the task, Generating InfoSphere DataStage jobs by using Rapid Generator.
  9. On the ABAP program setting page, specify the ABAP programs to use in the ABAP stage.
  10. On the Define Job Parameters page, ensure that the job parameters are correct and that each parameter set that you referenced exists in the project.
  11. On the Define column derivations page, you can optionally define an expressionDataStage Transformer for each InfoSphere DataStage type. These expressions are entered in Transformer stages of the generated jobs. You can cross-reference the input argument of the transformer with $1. Some examples are: TrimLeadingTrailing($1) or NullToEmpty($1).
  12. Click Next to start the job generation process. This process cannot be canceled.
  13. Click Finish.

Results

Jobs are generated on the InfoSphere DataStage server. If the Designer client is open, click Repository > Refresh to view the generated jobs.
