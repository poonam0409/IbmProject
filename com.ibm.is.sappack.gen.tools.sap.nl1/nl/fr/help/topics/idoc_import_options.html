Options d'importation d'IDoc

Cette page permet de configurer la manière dont les types d'IDoc sont importés dans un modèle de données logique. Chaque segment est mappé sur une table relationnelle et des zones supplémentaires permettent d'établir les relations parent à enfant de la hiérarchie des segments IDoc.

Zones de segment IDoc
Indique comment les différents types de segment IDoc sont mappés sur des attributs dans le modèle de données logique :
Définir toutes les zones de segment avec NULL admis
Place l'indicateur NULL admis sur toutes les zones.
Autoriser tous les types pour toutes les colonnes
Les types des attributs sont dérivés des types de données des zones de segment tels que VARCHAR ou DATE.
Utiliser VARCHAR uniquement
Le type VARCHAR est utiliser pour tous les attributs. Si vous sélectionnez cette option, vous pouvez définir un facteur de longueur par lequel la longueur d'origine est multipliée. Cette option est très utile si vous voulez utiliser les tables résultantes pour stocker temporairement les données dont la longueur excède la longueur maximale définie par SAP.
Option de table de vérification
Spécifiez des options d'extraction des tables de vérification connexes qui contiennent des valeurs admises pour certaines zones.
Importer les métadonnées pour les tables de vérification connexes
Importe toutes les tables de vérification des zones de segment IDoc importées.
Importer les métadonnées pour les tables de texte connexes
Importe la table de texte connexe de chaque table de vérification. Ces tables de texte contiennent des descriptions des valeurs des tables de vérification.
Liste noire des tables de vérification
Fichier contenant la liste des tables de vérification à éviter lors de l'importation de tables de vérification. Spécifiez un nom de table par ligne.
