IDoc-Importoptionen

Auf dieser Seite können Sie konfigurieren, wie IDoc-Typen in ein logisches Datenmodell importiert werden. Jedes Segment ist einer relationalen Tabelle zugeordnet und zusätzliche Felder werden verwendet, um die übergeordnete und untergeordnete Beziehung der IDoc-Segmenthierarchie zu erstellen.

IDoc-Segmentfelder
Geben Sie an, wie unterschiedliche Typen von IDoc-Segmentfeldern Attributen im logischen Datenmodell zugeordnet werden.
Make all segment fields nullable
Setzt die Markierung nullable flag für alle Felder.
Allow all types for all columns
Die Attributtypen werden aus den Datentypen der Segmentfelder abgeleitet, wie z. B. VARCHAR oder DATE.
Use VARCHAR only
Verwendet VARCHAR als Typ für alle Attribute. Wenn diese Option ausgewählt ist, können Sie einen Längenfaktor definieren, mit dem die Originallänge multipliziert wird. Diese Option ist hilfreich, wenn Sie Daten, deren Länge die von SAP definierte maximale Länge überschreitet, temporär in Ergebnistabellen speichern wollen.
Check table options
Geben Sie Optionen zum Extrahieren zugehöriger Prüftabellen an, die gültige Werte für bestimmte Felder enthalten.
Metadaten für zugehörige Prüftabellen importieren
Importieren Sie alle Prüftabellen für die importierten IDoc-Segmentfelder.
Metadaten für zugehörige Texttabellen importieren
Importieren Sie die zu jeder Prüftabelle zugehörige Texttabelle. Diese Texttabellen enthalten Beschreibungen der Werte von Prüftabellen.
Schwarze Liste für Tabellen überprüfen
Eine Datei mit einer Liste von Prüftabellen, die beim Importieren der Prüftabellen umgangen werden sollen. Geben Sie einen Tabellennamen pro Zeile ein.
