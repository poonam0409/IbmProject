Installation and Prerequisites

The IBM InfoSphere Information Server Packs for SAP Applications consist of two components (see also here):
  1. Client:
    The client is an extension of the InfoSphere Data Architect and is installed with its own launchpad using IBM Installation Manager.
  2. Server:
    For the Rapid Generator it is required to install a server component that performs the actual job generation. This server component is an enterprise application (provided as an .EAR file) that is installed on the WebSphere Application Server (WAS) of IBM Information Server. The .EAR file can be installed with standard WAS means, e.g., through the administrative console using a web browser.

Client install

The client install installs the Rapid Modeler as well as the Rapid Generator client.
  1. Copy the installation package to the machine where IBM InfoSphere Data Architect is installed.
  2. Run the command launchpad.exe of the installation package
  3. Click through the wizard and make sure that you use the same package group where you installed InfoSphere Data Architect for the installation.

Server component install

The server component install is only required for Rapid Generator.
There are a couple of ways to install the enterprise applications in WebSphere Application Server. One is through the administrative console using a web browser:
  1. Find out the port of the administrative console. Typically, the following works: Open the Information Server Web Console and take a look at the URL it points to. If this number is 9080 (default port) the administrative console typically runs on port 9060. If it is 9081, the admin console runs on 9061 and so on.
  2. Point your web browser to http://<InformationServerDomainHost>:9060/ibm/console.
  3. Login using the Application Server administrator credentials.
  4. Open the Enterprise Applications menu on the left.
  5. Click on Install
  6. Click on Browse and select the com.ibm.is.sappack.gen.server.ear file of the installation package
  7. Walk through the wizard leaving all default settings
  8. After the installation is installed save the master configuration.
  9. In the Enterprise Applications view browse to the application. It should have a red cross next to it as a sign that the application is not yet started.
  10. Check the checkbox next to the application and click on Start. A green arrow should now appear next to the application.
