IDoc Extract ジョブのジョブ生成の構成

IBM® InfoSphere™ Rapid Generator for SAP ウィザードを使用して、IDoc Extract ジョブを生成することができます。

始める前に

  • IBM InfoSphere Data Architect Rapid Modeler for SAP および IBM InfoSphere Rapid Generator for SAP で使用するためのワークスペースを構成します。
  • 「InfoSphere DataStage® 環境を構成します」が正しく構成されている必要があります。
  • SAP メタデータを論理データ・モデルにインポートし、その論理データ・モデルを物理データ・モデルに変換します。

手順

  1. 「物理データ・モデル」(.dbm ファイル) を右クリックし、「Rapid Generator for SAP」 > 「DataStage ジョブの生成」を選択します。
  2. 「ジョブ・タイプの選択」ページで、「IDoc Extract」を選択します。
  3. 「Information Server の詳細」ページで、InfoSphere Information Server インストールへの接続情報を入力し、ウィザードの実行時に作成されたすべてのジョブの接頭部を提供します。 接続の詳細が、InfoSphere Information Server クライアント・アプリケーション (InfoSphere DataStage and QualityStage™ デザイナー クライアントなど) を使用する場合に入力された内容と同じであることを確認してください。
  4. 「次へ」をクリックして、接続の詳細を検証します。
  5. 「ソース・オプション」に、IDoc の処理元の SAP システムに関する情報を指定します。
  6. 「ターゲット・オプション」ページで、SAP 論理テーブルのターゲットとして生成されるステージを構成します。
  7. 「SAP 設定の検証」をクリックして、このページにある SAP 関連の設定をすべて検証します。検証結果は新しいウィンドウに表示されます。 「Rapid Generator を使用した InfoSphere DataStage ジョブの生成」タスクを続行します。
  8. 「ジョブ・パラメーターの定義」ページで、ジョブ・パラメーターが正しいこと、および参照した各パラメーター・セットがプロジェクトに存在することを確認します。
  9. 「列出力仕様の定義」ページで、InfoSphere DataStage タイプの それぞれに「DataStage Transformer」式をオプションで定義できます。 これらの式は、生成されるジョブの Transformer ステージに入力されます。トランスフォーマーの入力引数は $1 を使用して相互参照できます。例えば、TrimLeadingTrailing($1) や NullToEmpty($1) です。
  10. 「次へ」をクリックして、ジョブ生成プロセスを開始します。このプロセスはキャンセルできません。
  11. 「完了」をクリックします。

タスクの結果

ジョブが InfoSphere DataStage サーバー上に生成されます。 Designer Clientがオープンしている場合は、「リポジトリー」 > 「最新表示」をクリックして、生成されたジョブを表示します。
