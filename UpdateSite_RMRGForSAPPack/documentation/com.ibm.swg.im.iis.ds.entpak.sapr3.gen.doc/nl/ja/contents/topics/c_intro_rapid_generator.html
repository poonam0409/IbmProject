IBM InfoSphere Rapid Generator for SAP Applications の概要

IBM® InfoSphere™ Rapid Generator for SAP Applications は、SAP との間でデータの抽出やロードを行う InfoSphere DataStage® ジョブを InfoSphere Data Architect 内の物理データ・モデルから生成するウィザードです。 この物理データ・モデルは、IBM InfoSphere Rapid Modeler for SAP を使用して、SAP メタデータをインポートした論理データ・モデルを変換することによって、作成する必要があります。

SAP を含む大規模な ETL プロジェクトは多くの場合、複雑で、長期間にわたって行われます。時間が経つにつれて、多くの場合、SAP からの抽出と SAP へのロードの要件は変化します。このような変化により、包括的な ETL ジョブにより伝搬されなければならない一連の変更がトリガーされます。 InfoSphere Information Server ツールである Impact Analysis は、これらの変更の管理に役立ちます。多くの場合、開発者は引き続き多数のジョブを変更する必要があります。ジョブが IBM InfoSphere Rapid Generator for SAP を使用して生成される場合、論理データ・モデルの変更には、物理データ・モデルが必要であり、それに関連したジョブが再生成される必要があります。 古い既存のジョブが上書きされます。

IBM InfoSphere Rapid Generator for SAP ウィザードは、IBM InfoSphere Information Server インスタンスに接続し、サーバー上にジョブを生成します。作成されたジョブを処理するには、InfoSphere DataStage and QualityStage™ デザイナー クライアントか InfoSphere DataStage and QualityStage ディレクター クライアントのどちらかを使用できます。

InfoSphere DataStage ジョブで IBM InfoSphere Information Server Pack for SAP Applications を使用する場合、SAP から抽出したいテーブルごとに 1 つの ABAP ステージを構成する必要があります。テーブルの長いリストがある場合、同じ設定を使用して多数の ABAP ステージを手動で構成する必要があります。Rapid Generator for SAP を使用すると、ウィザードに設定値を入力することで、1 組のジョブ全体を生成することができます。これらの設定を 1 回入力するだけで済みます。 ジョブが生成された後、ABAP ステージをそのまま、正しく構成されたままにして、それらのジョブを引き続き処理することができます。

作成されるジョブの種類は、IBM InfoSphere Rapid Modeler for SAP を使用してインポートされたメタデータによって異なります。以下のジョブ・タイプが、IBM InfoSphere Rapid Generator for SAP ウィザードで使用可能です。

ABAP Extract ジョブ
論理テーブルを論理データ・モデルにインポートした場合は、それらのテーブルを SAP から抽出して、物理データ・モデルで記述されるテーブルに保管するジョブを生成できます。
IDoc Extract ジョブおよび IDoc Load ジョブ
IDoc セグメントを論理データ・モデルにインポートした場合は、以下の機能を持つジョブを作成するかどうかを選択できます。
  • SAP から InfoSphere DataStage に送信された IDoc を処理し、それを物理データ・モデルのデータベースに書き込むジョブ (IDoc Extract)。
  • IDoc を構成するデータベースからデータを読み取り、そのデータを SAP に送信するジョブ (IDoc Load)。
