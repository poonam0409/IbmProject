SAP システムとモデルを調整する

論理データ・モデルに SAP メタデータをインポートした後、そのメタデータは、インポート時点における SAP システムのスナップショットを表します。新規モデルに対して IBM® InfoSphere™ Data Architect Rapid Modeler for SAP のメタデータ・インポート・ウィザードを再実行することによって、モデルを再抽出または調整したい場合があります。新規モデルが作成された後は、InfoSphere Data Architect を使用して、元の論理データ・モデルと新規論理データ・モデルを比較できます。メタデータの再抽出では、すべてのチェック・テーブルとテキスト・テーブルが再収集されます。

始める前に

IBM InfoSphere Data Architect Rapid Modeler を使用して SAP メタデータが論理データ・モデルにインポートされたことを確認します。

このタスクについて

メタデータの調整が便利なシナリオのいくつかは、次のとおりです。
  • 論理データ・モデルと、同じ SAP システム内のモデルとを比較して、モデルが変更されたか確認します。例えば、新規バージョンにアップグレードした後や開発サイクルの終了時後に、モデルが変更される可能性があります。
  • SAP システム同士を比較します。例えば、テスト・システムとそれに対応する実動システム間や 2 つの別々の SAP システム間で、相違が検出される場合があります。

手順

  1. SAP メタデータを含んでいる論理データ・モデル (.ldm ファイル) を右クリックし、「Rapid Modeler for SAP」 > 「メタデータを新規モデルに再抽出」を選択します。
  2. 「前に実行したインポーター」ページで、IBM InfoSphere Data Architect Rapid Modeler for SAP で実行した前のインポート実行を選択し、「次へ」をクリックします。
  3. 作成する新規論理データ・モデルの名前を入力します。.ldm ファイル拡張子は付けません。
  4. メタデータを再インポートする元の SAP システムを選択します。
  5. 「再抽出の開始」をクリックします。
  6. 「OK」をクリックします。 元のインポートのオプションを含む IBM InfoSphere Data Architect Rapid Modeler for SAP インポート・ウィザードが開始します。
  7. 2 つの論理データ・モデルを比較するには、モデルを右クリックし、「比較」 > 「相互比較」を選択します。

タスクの結果

新規論理データ・モデルが作成されました。このモデルには再調整された SAP メタデータが含まれます。
