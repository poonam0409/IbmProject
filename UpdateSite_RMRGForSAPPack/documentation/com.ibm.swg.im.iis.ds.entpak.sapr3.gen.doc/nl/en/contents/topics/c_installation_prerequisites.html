IBM InfoSphere Data Architect components for Pack for SAP Applications, Version 9.1.*
Feedback

Prerequisites

InfoSphere™ Rapid Modeler for SAP and InfoSphere Rapid Generator for SAP client is an extension of the IBM InfoSphere Data Architect and is installed with its own launchpad by using IBM Installation Manager. For InfoSphere Rapid Generator for SAP, you must install a server component that generates the jobs. This server component is an enterprise application that is installed on the WebSphere® Application Server of InfoSphere Information Server. The application is provided as a .ear file and can be installed with standard WebSphere Application Server, for example, through the administrative console by using a web browser.

Client prerequisites

InfoSphere Rapid Modeler for SAP and InfoSphere Rapid Generator for SAP client is supported only on Windows.

Table 1 shows the prerequisite products and components that must be available on the client system.
Table 1. Prerequisites for client installation
Product Version Remarks
InfoSphere Data Architect 8.5 and 9.1.* Required
InfoSphere Information Server client 11.* Required only for InfoSphere Rapid Generator for SAP
IBM InfoSphere Information Server Pack for SAP Applications client 8.0.0.* Required only for InfoSphere Rapid Generator for SAP
SAP JCo 3 package 3.0.x Download the package from SAP and ensure that sapjco3.jar and sapjco3.dll files are available in a directory.

Server Prerequisites

InfoSphere Rapid Modeler for SAP does not require an InfoSphere Information Server installation. The InfoSphere Rapid Generator for SAP is supported only on Windows and AIX® platforms of InfoSphere Information Server.

Table 2. Prerequisites for server installation
Product Version
InfoSphere Information Server 11.*
IBM InfoSphere Information Server Pack for SAP Applications server 8.0.0.*

Prerequisites for the SAP environment

This configuration contains user authorization profiles for design and run time, and a function module to be installed on the SAP system. Follow the steps that are required for the IBM InfoSphere Information Server Pack for SAP Applications. The Pack installation contains transport files that you can import into your SAP system. Alternatively, you can configure your SAP system manually.

Note:
  • To use the InfoSphere Rapid Modeler for SAP, it is sufficient that the SAP user has the necessary design time authorizations.
  • To use InfoSphere Rapid Generator for SAP only for extracting and loading IDocs (no table extracts), it is sufficient that the SAP user has the authorizations for IDoc processing.
  • If the user has SAP_ALL privileges, no further configuration is required.

Supported Database

InfoSphere Rapid Generator for SAP works only on physical data models that are created for IBM DB2® Database for Linux, UNIX, and Windows and Oracle.
