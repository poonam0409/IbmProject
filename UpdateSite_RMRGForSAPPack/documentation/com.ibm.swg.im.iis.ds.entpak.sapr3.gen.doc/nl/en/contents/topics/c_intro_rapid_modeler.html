Introduction to InfoSphere Rapid Modeler for SAP Applications

InfoSphere™ Rapid Modeler for SAP Applications is a wizard that imports metadata from an SAP system into a logical data model of IBM InfoSphere Data Architect.

The following kinds of metadata can be imported:

Logical tables
An SAP system stores its business data in tables that are accessed by using Advanced Business Application Programming (ABAP) code. For example, KNA1, KNB1,and KNVV tables contain a part of the customer master data. The MARA and MARC tables contain data for material master data. IBM InfoSphere Data Architect Rapid Modeler for SAP identifies a list of tables in an SAP system. These tables are then imported into a logical data model by creating an entity for each SAP table and respective attributes for each column.
IDocs
An IDoc in SAP is a hierarchical and nonrelational data structure that is used to communicate with other systems. IDocs have a type that defines the structure of the hierarchy. Each hierarchy level is called a segment. For example, the DEBMAS IDoc type represents the debitor (customer) business object. It contains the E1KNA1M segment, which represents basic customer data. By using IBM InfoSphere Data Architect Rapid Modeler for SAP, you can browse an SAP system for IDoc types and segments. Each IDoc segment is mapped to an entity, and the IDoc fields are mapped to attributes. The hierarchical structure is modeled by technical attributes for each entity that represents the hierarchy.
