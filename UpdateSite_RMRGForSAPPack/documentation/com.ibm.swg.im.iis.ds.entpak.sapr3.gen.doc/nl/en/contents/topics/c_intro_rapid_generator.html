Introduction to InfoSphere Rapid Generator for SAP Applications

InfoSphere™ Rapid Generator for SAP Applications is a wizard that generates InfoSphere DataStage® jobs that extract and load data from and to SAP from a physical data model in InfoSphere Data Architect. The physical data model must be created by using the InfoSphere Rapid Modeler for SAP to transform the logical data model into which the SAP metadata is imported.

Large ETL projects that involve SAP are often complex and take place over a longer period. Over time frame, the requirements for extracting and loading from or to SAP often change. Such changes trigger a chain of changes that must be propagated through extensive ETL jobs. Impact Analysis, an InfoSphere Information Server tool, helps in managing these changes, often a developer must still modify numerous jobs. If jobs are generated by using InfoSphere Rapid Generator for SAP, a change in the logical data model requires the physical data model, and the jobs associated with it to be regenerated. Existing jobs that are out-of-date are overwritten.

InfoSphere Rapid Generator for SAP wizard connects to an IBM InfoSphere Information Server instance and generates the jobs on the server. You can use either the InfoSphere DataStage and QualityStage™ Designer client or the InfoSphere DataStage and QualityStage Director client to work with the created jobs.

When you use the IBM InfoSphere Information Server Pack for SAP Applications in an InfoSphere DataStage job, you must configure one ABAP stage for each table that you want to extract from SAP. If you have a long list of tables, you must configure many ABAP stages manually with the same settings. You can use the Rapid Generator for SAP to generate a whole set of jobs by entering the settings in the wizard. You must enter these settings only once. After the jobs are generated you can continue to work with them, leaving the ABAP stages untouched and thus configured correctly.

The kinds of jobs that are created depend on the metadata that was imported with the InfoSphere Rapid Modeler for SAP. The following job types are available in the InfoSphere Rapid Generator for SAP wizard:

ABAP Extract jobs
If you imported logical tables into your logical data model, you can generate jobs that extract those tables from SAP and store them in tables that are described by the physical data model.
IDoc Extract and IDoc load jobs
If you imported IDoc segments into a logical data model, you can select whether you want to create jobs that have the following functions:
  • Process IDocs that are sent from SAP to InfoSphere DataStage and write them into the database of your physical data model (IDoc Extract).
  • Read data from the database construct IDocs and send the data to SAP (IDoc Load).
