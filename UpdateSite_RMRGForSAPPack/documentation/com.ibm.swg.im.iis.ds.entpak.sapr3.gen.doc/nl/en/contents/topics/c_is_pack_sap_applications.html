Overview

As part of almost any SAP project, you must identify, analyze, and document the data model of SAP. The data models are almost always customized and augmented, and the custom data models differ from the standard models. Extracting data from and loading data into an SAP system requires knowledge of the external applications and the SAP interfaces. In enterprises, different teams are responsible for data integration and the SAP system. This division of responsibility leads to improvised integrations, for example, on the exchange of flat files.

When you integrate external applications with an SAP system, two challenges arise:

  • Where is the data that is relevant for the task in the SAP system? What data structures in SAP store which information?
  • How do I extract and load this data to and from my external application? Which extraction or loading mechanisms are available and how do I use them?

The challenges are not independent. For example, SAP IDocs define both a data structure and a transport mechanism. A transport mechanism is not the same as the mechanisms that are typically used to extract SAP logical tables. The following tools address the issues:

InfoSphere™ Rapid Modeler for SAP Applications
You can use this tool to browse the SAP data model and import all the tables that are related through foreign key relationships that is created by using IBM InfoSphere Data Architect. You can create diagrams from the tables that you import, and add comments and annotations to better understand the metadata.
InfoSphere Rapid Generator for SAP Applications
You can use this tool to generate InfoSphere DataStage® jobs with the SAP metadata that was imported by using InfoSphere Rapid Modeler for SAP applications. These InfoSphere DataStage jobs extract or load the actual data of the SAP objects described by the metadata that was imported to or from a relational database or flat files.
You can move data from various source systems such as, flat files, relational databases, or other SAP systems, into a single, consolidated SAP system. During this process, you build an intermediate database which is sometimes called as the staging area. You can use this database to store all data to be loaded into the target system. In addition, complex normalization and cleansing operations are performed here. You can use the InfoSphere Rapid Modeler and the InfoSphere Rapid Generator for SAP to:
  • Create the schema of the staging area for the SAP target system.
  • Generate jobs that load the target SAP system from the staging area.
  • Create a schema and generate extract jobs for SAP source systems.

Both the InfoSphere Rapid Modeler for SAP Applications tool and the InfoSphere Rapid Generator for SAP Applications tool have clients that are implemented as extensions (Eclipse plugins) to InfoSphere Data Architect. Both tools also rely on the SAP JCo library, which provides connectivity to an SAP system. InfoSphere Rapid Generator for SAP Applications requires InfoSphere Information Server to generate and compile the InfoSphere DataStagejobs. An enterprise application is deployed as a .ear file in the same instance of WebSpehere Application Server that InfoSphere Information Server uses.

Figure 1. Architecture of IBM InfoSphere Information Server Pack for SAP Applications
  • Introduction to InfoSphere Rapid Modeler for SAP Applications
    IBM InfoSphere Data Architect Rapid Modeler for SAP Applications is a wizard that extracts metadata from an SAP system into a logical data model of IBM InfoSphere Data Architect.
  • Introduction to InfoSphere Rapid Generator for SAP Applications
    InfoSphere Rapid Generator for SAP Applications is a wizard that generates InfoSphere DataStage jobs that extract and load data from and to SAP from a physical data model in InfoSphere Data Architect. The physical data model must be created by using the InfoSphere Rapid Modeler for SAP to transform the logical data model into which the SAP metadata is imported.
