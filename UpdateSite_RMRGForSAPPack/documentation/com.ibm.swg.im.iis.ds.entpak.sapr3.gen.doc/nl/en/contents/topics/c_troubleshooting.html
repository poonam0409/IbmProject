Troubleshooting

The troubleshooting topic addresses issues that you might encounter with the IBM® InfoSphere™ Information Server Pack for SAP Applications.

Client

If you encounter an issue while using the Rapid Modeler for SAP and, or Rapid Generator for SAP, check the .log file in the workspace's .metadata directory for errors.

Server

For Rapid Generator for SAP, you might also want to check the log files on the WebSphere® Application Server where you have installed the server component enterprise application. This directory can typically be found here: <WAS install directory>/AppServer/profiles/default/logs/server1. Start with the SystemOut.log and SystemErr.log files. If you have enabled tracing, then you will also see a trace.log file.

To switch on the diagnostic trace see the respective page in the WebSphere Application Server help. The name of the trace component is com.ibm.is.SAPPacks.

Examples

There are a couple of examples related to the IBM InfoSphere Information Server Pack for SAP Applications. They require an existing Data Design Project.

To create the examples, go to File > New > Examples > IBM InfoSphere Information Server Pack for SAP Applications.
  • Business Object example model: This is an example for a predefined SAP model that can be used by the Rapid Modeler for SAP.
  • Example file for check table blacklists used in the Rapid Modeler for SAP. The Rapid Modeler for SAP can use files that contain the name for tables that are ignored during import. See Importing SAP metadata with InfoSphere Data Architect Rapid Modeler for SAP. This is an example how this file might look like.
  • Example file for RFC destinations used in the Rapid Generator for SAP for ABAP extract jobs

    To generate jobs by using the ABAP extract stage with the RFC transfer method, one must provide a list of RFC destinations and associated program IDs in a file. This is an example how this file might look like.

Known Issues

This section contains a description of known issues. For known issues and help on the IBM InfoSphere Information Server Pack for SAP R/3 in InfoSphere DataStage®, see Pack for SAP R/3, Version 6.5.0.1: Known problems.
  1. Issue: A generated IDoc load or extract job ends with one of the following error messages:
    • Fatal Error: Fatal: IDOC_F_00131`:`Required property 'DEFAULTUSERNAME' not found in configuration file.
    • Fatal Error: Fatal: IDOC_F_00131`:`Required property 'SAPAPPSERVER' not found in configuration file.

    Solution: Check that the provided connection name does exist and is correctly configured for your DataStage project.

  2. Issue: SAP JCo library cannot be loaded.

    Solution: See IBM technote 1418488. This technote is written for a different product but the solution applies here also.

  3. Issue: Rapid Generator for SAP does not generate jobs for a physical model other than IBM DB2® Database for Linux, UNIX, and Windows.

    Reason: Currently only physical data models for DB2 for Linux/Unix/Windows are supported with Rapid Generator for SAP.
