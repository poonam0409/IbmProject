IBM InfoSphere Data Architect components for the Pack for SAP Applications

The IBM® InfoSphere™ Information Server Pack for SAP Applications offers InfoSphere Data Architect tools for importing logical data models and generating InfoSphere DataStage® jobs that use the imported metadata.

  • Overview
    As part of almost any SAP project, you must identify, analyze, and document the data model of SAP. The data models are almost always customized and augmented, and the custom data models differ from the standard models. Extracting data from and loading data into an SAP system requires knowledge of the external applications and the SAP interfaces. In enterprises, different teams are responsible for data integration and the SAP system. This division of responsibility leads to improvised integrations, for example, on the exchange of flat files.
  • Prerequisites
    InfoSphere Rapid Modeler for SAP and InfoSphere Rapid Generator for SAP client is an extension of the IBM InfoSphere Data Architect and is installed with its own launchpad by using IBM Installation Manager. For InfoSphere Rapid Generator for SAP, you must install a server component that generates the jobs. This server component is an enterprise application that is installed on the WebSphere® Application Server of InfoSphere Information Server. The application is provided as a .ear file and can be installed with standard WebSphere Application Server, for example, through the administrative console by using a web browser.
  • Installing the IBM InfoSphere Data Architect components
    You must install the client and server components for the InfoSphere Rapid Modeler and the InfoSphere Rapid Generator for SAP.
  • Configuring the IBM InfoSphere DataStage environment for Rapid Generator for SAP
    Before you use InfoSphere Rapid Generator for SAP to generate jobs, you must configure the IBM InfoSphere DataStage environment.
  • Preparing the InfoSphere Data Architect workspace
    Before you use the InfoSphere Rapid Modeler and the InfoSphere Rapid Generator for SAP, you must prepare the InfoSphere Data Architect workspace.
  • Importing SAP metadata by using Rapid Modeler
    You can use the InfoSphere Rapid Modeler for SAP wizard to import either the metadata from logical tables or IDoc types from SAP into a logical data model.
  • Generating InfoSphere DataStage jobs by using InfoSphere Rapid Generator for SAP
    You can use the InfoSphere Rapid Generator for SAP wizard to generate ABAP extract, IDoc extract, and IDoc load jobs that extract or load data from and to an SAP system.
  • Reconciling your model with an SAP system
    After you imported SAP metadata into a logical data model, the metadata represents a snapshot of the SAP system at that moment. Sometimes you might want to re-import or reconcile your model by running InfoSphere Rapid Modeler for SAP metadata import wizards again on a new model. After the new model is created, you can use InfoSphere Data Architect to compare the original and the new logical data model. The metadata re-import recollects all check and text tables.
  • Troubleshooting
    The troubleshooting topic addresses issues that you might encounter with the IBM InfoSphere Information Server Pack for SAP Applications.
