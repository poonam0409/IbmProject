IDoc Load

Specify information required to load IDocs to SAP.

Enable IDoc Load status
Select the check box to generate the IDoc load jobs with additional stages that generate, assign, and save IDs for the IDocs to be loaded. The IDs are generated by using the surrogate key mechanism of InfoSphere® DataStage® and are assigned to the control record of IDocs. When IDocs are loaded to the SAP system, the IDs are saved to the Conversion Workbench database. You can verify if the IDocs sent to SAP were processed successfully by comparing the IDs in the database with the IDs in the SAP system.
If this option is not enabled, the IDoc load stage cannot verify if the IDocs were processed successfully as the IDoc mechanism is asynchronous and the processing might happen a long time after the IDocs were loaded.
Surrogate key source file on DataStage server
Specify the location of surrogate key source file on the InfoSphere DataStage server that is required to generate the IDs.
Name of the corresponding business object
Specify name of the business object to extract IDocs from. The specified name and the generated ID are copied to the Conversion Workbench database to enable filtering to verify if the IDocs were loaded successfully.
