ABAP Transfer Methodoptions

Use this page to configure the ABAP extract options in the job generation design.

Note: You can use InfoSphere™ DataStage® job parameters for text fields by enclosing the text with the "#" symbol. The parameter sets are indicated with the syntax "#" #ParameterSetName.ParameterName#. For more information about parameters and parameter sets, see the IBM® InfoSphere DataStage and QualityStage™ Designer Client Guide
RFC transfer method options
Specify the RFC transfer method options:
SAP Gateway Host
Host name of the SAP gateway offering services to communicate with the SAP server.
SAP Gateway Service
Name of the gateway service.
RFC Destinations
A simple text file where each line consists of two entries separated with a blank space. The first entry is the name of the RFC destination configured on the SAP system. The second entry is the corresponding program ID of the RFC destination.
Max. Number of Flows in a Job
ABAP extract jobs can be generated with multiple ABAP stages in order to reduce the number of jobs. However, in an ABAP extract job, each ABAP stage requires a unique RFC destination to be configured in SAP. Therefore, the maximum number of ABAP stages is limited by the number of RFC destinations provided. It also defines the maximum number of flows per job, or ABAP stage instances within a single IBM InfoSphere DataStage job, limited to the number of RFC destinations provided.
Create RFC destination automatically
Select this option if the RFC destination is to be created at run time by the IBM InfoSphere DataStage job.
Delete RFC destination if necessary
Select this option to allow deletion of RFC destination if required.
Validate SAP Settings
Click this button to validate the SAP connection settings.
